    The #26 epic in this map

    The fee-discovery redesign (#26) was delivered by a chain of children — all merged; the epic is complete — near-sequential because the later ones consume the earlier ones:

    T28["#28 FeeTag<br/>metadata codec ✅"] --> T29["#29 Store<br/>allowance model ✅"]
    T29 --> T30["#30 FeeIndexer<br/>chain-follower ✅"]
    T29 --> T31["#31 Publish rewrite<br/>+ fee-status ✅"]
    T30 --> T32["#32 devnet<br/>publish smoke ✅"]
    T31 --> T32
    classDef done fill:#1b4332;
    class T28,T29,T30,T31,T32 done;
    • #28 FeeTag (PR #33) — the agreement surface: the exact on-chain bytes both the indexer and the publish gate depend on.
    • #29 Store (PR #34) — the rollback-aware allowance model the indexer writes and the publish gate reads.
    • #30 FeeIndexer (PR #36) — the chain-follower that populates allowances and records malformed/untagged payments.
    • #31 Publish (PR #37) — retired the datum gate, admits on the allowance, serves the fee-status evidence query. The datum path and plutus-ledger-api are now gone entirely.
    • #32 devnet publish smoke (PR #38) — the mandatory live-boundary test, now a required CI job: it boots a forging Conway devnet and drives a real metadata-tagged payment through fee_not_seen → fee_unconfirmed → ready_to_publish → 201 → receipt, so a mock-green publish path can never merge again (the datum DOA is what it guards against).

    Technology constraints

    • Haskell + Nix flake, matching the cardano-tx-tools / amaru-treasury-tx toolchain; crypto and phase-1 reused from those, never forked, never delegated to JavaScript.
    • Chain access is N2C to a local cardano-node by default, behind an interface additional sources may implement.
    • State is an explicit RocksDB persistence layer, swappable behind an interface, durable across restart.
    • API is contract-first: the checked-in OpenAPI 3.1 document is the source of truth and the interop standard operators implement.
